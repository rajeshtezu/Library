## SIP

- Mostly for VoIP phone call

Format:

```
...
From: Admin <sip:admin@net.com>;
To: Bob <sip:bob@bobtech.com>;
...
```

- Response codes are same as http response codes
- Intermediate server: PBX
- SIP does not negotiate media type or codecs

## SDP

- Works hand in hand with `SIP`
- Negotiate media type and codecs
- Extra information sent with some SIP messages
- SDP does not need to be only used with SDP, it can be used with http or other signaling as well

**Early Offer**: If `SDP` is included in call invite messages

**Delayed Offer (Aka Late Offer)**: If `SDP` is not included in the invite messages

## RTP & RTCP

**RTP**: Real-time Transport Protocol

**RTCP**: RTP Control Protocol

- RTP is popular media protocol
- Carries actual audio and video
- RTP commonly uses UDP traffic
- RTCP monitors call quality and manages call and used along with RTP
- RTP is also used in WebRTC SDP

## FMTP

Flight Message Transfer Protocol

- Communication stack based on the TCP/IP
- Run on top of a multicast network service for delivery of continuously generated files
- Also used in WebRTC SDP

## DTMF

Dual Tone Multi Frequency

- Sounds or tones generated by a telephone when the numbers are pressed
- These tones are transmitted with the voice channel
- DTMF is used to control automated equipment and signal user intent, such as the number they wish to dial
- telephone-event -> in SDP

## JSEP

Javascript Session Establishment Protocol

- Abstract all the inner working of SDP
